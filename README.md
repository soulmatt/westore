# WeStore - 통합 배송 관리

다중 마켓플레이스 발주서를 하나의 택배 업로드 파일로 합치고, 택배사 송장을 각 마켓플레이스 발송처리 파일로 변환하는 브라우저 도구입니다.

**서버 없음** — 모든 데이터는 브라우저에서만 처리되며 외부로 전송되지 않습니다.

## 사용법

### 1단계: 발송처 정보 입력

발송처명, 전화번호, 주소를 입력합니다. "브라우저에 저장"으로 다음 방문 시 자동 로드됩니다.

### 2단계: 발주서 합치기 (Job1)

1. 각 마켓플레이스에서 다운로드한 발주서 엑셀 파일을 드롭존에 올립니다
2. 마켓플레이스가 자동 감지되어 파싱됩니다
3. **"1번 파일 다운로드"** 클릭 → 택배 프로그램에 업로드할 통합 엑셀 파일 생성

### 3단계: 송장 입력 (Job2)

1. 택배 프로그램에서 내려받은 송장 파일을 드롭존에 올립니다
2. 주문번호 기반으로 송장번호가 자동 매칭됩니다
3. **"2번 파일 다운로드"** 클릭 → 각 마켓플레이스별 발송처리 엑셀 파일 생성

## 실행

정적 파일 서버로 실행하거나 GitHub Pages로 배포합니다.

```bash
# 로컬 실행 예시
npx serve .
```

## 택배사 변경

`js/core/vendor-config.js`의 `CURRENT_VENDOR_ID` 값을 변경합니다:

```js
const CURRENT_VENDOR_ID = 2; // 1: CJ대한통운, 2: 롯데택배
```

## 택배 양식 컬럼 변경

택배 업로드 파일(Job1)의 열 이름과 순서는 `js/marketplaces/base-marketplace.js`의 `convertToInvoiceFormat()` 메서드에서 코드로 관리합니다.
